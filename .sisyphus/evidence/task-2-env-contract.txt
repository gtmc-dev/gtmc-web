## Task 2 — Features Env Contract (Happy Path)
Date: 2026-03-13

### Variable Ownership Map

| Variable                    | Owner Module      | Purpose                                  | Action      |
|-----------------------------|-------------------|------------------------------------------|-------------|
| FEATURES_GITHUB_REPO_OWNER  | features only     | Target repo owner for issue operations   | NEW (add)   |
| FEATURES_GITHUB_REPO_NAME   | features only     | Target repo name for issue operations    | NEW (add)   |
| GITHUB_SYSTEM_PAT           | features + admin  | GitHub API authentication token          | UNCHANGED   |
| GITHUB_REPO_OWNER           | admin only        | Repo owner for workflow dispatch         | UNCHANGED   |
| GITHUB_REPO_NAME            | admin only        | Repo name for workflow dispatch          | UNCHANGED   |

### Contract Rules
1. lib/github-features.ts reads FEATURES_GITHUB_REPO_OWNER and FEATURES_GITHUB_REPO_NAME ONLY.
2. actions/admin.ts reads GITHUB_REPO_OWNER and GITHUB_REPO_NAME ONLY (unchanged).
3. GITHUB_SYSTEM_PAT is shared — used by both modules, no change.
4. Strict no-fallback: lib/github-features.ts MUST NOT contain any fallback expression like:
       process.env.FEATURES_GITHUB_REPO_OWNER || process.env.GITHUB_REPO_OWNER
   or any equivalent.
5. Misconfiguration in Features module throws GithubFeaturesError with code CONFIG_MISSING, listing new env key names.

### No Fallback Ban (verbatim)
BANNED: || process.env.GITHUB_REPO_OWNER  (in Features config path)
BANNED: || process.env.GITHUB_REPO_NAME   (in Features config path)
